# Contributing

Thanks for helping keep this Kubernetes release logo collection up to date! This is a small repo, so the guidance is short.

## Adding a new release logo

1. Download the official SVG/PNG logo from the Kubernetes release blog post or the [`sig-release`](https://github.com/kubernetes/sig-release/tree/master/releases) repository.
2. Commit the file as `vX.YY.<ext>` (e.g. `v1.36.svg`) at the repo root.
3. Add a new entry at the **top** of `README.md`:
   ```markdown
   ## vX.YY - <Codename>
   ![<Codename>](vX.YY.<ext>)
   ```

## Before opening a PR

- **Rebase on the latest `main`.** New entries belong at the top of the README; if your branch is based on an older `main`, the diff may insert your entry in the wrong position.
- **Keep the PR focused** — one logo per PR. Unrelated cleanups (header normalization, `.gitignore` tweaks, etc.) should go in separate PRs so the history stays easy to follow.
- **Review the diff yourself before submitting**, even (especially) if the PR was generated by an AI coding tool. Stray changes that aren't about the logo should be removed before opening the PR.

## A note on AI-assisted contributions

AI coding tools (Codex, Copilot, etc.) are welcome — but please make sure your commits are **authored by you**, not by the tool's bot account.

When an AI tool commits on your behalf, the commits are often authored as `copilot-swe-agent[bot]`, `Claude`, or similar. After a merge, those bot identities can leak into this repo's Contributors list and cannot easily be removed (it requires a manual cache rebuild by GitHub Support).

Before pushing, please re-author any AI-generated commits to yourself:

```bash
git commit --amend --reset-author --no-edit
# or for multiple commits:
git rebase -i <base> --exec 'git commit --amend --reset-author --no-edit'
```

Then verify with `git log --format='%an <%ae>' -n 5` that you (not a bot) are listed as author. Thanks!
